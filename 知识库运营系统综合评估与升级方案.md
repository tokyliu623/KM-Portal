知识库运营系统综合评估与升级方案
一、问题诊断
P0 Bug：类型不匹配导致 Token 查找失败
位置：server.ts:244
// 问题代码
return db.tokens.find((t) => t.kb_id === String(kbId) && t.status === 'active') || null;
根因：
- token-store.json 存储 kb_id: 13469（数字）
- 比较时使用 String(kbId) → "13469"
- 13469 === "13469" → false
修复方案：
// 方案1：统一转为字符串比较
return db.tokens.find((t) => String(t.kb_id) === String(kbId) && t.status === 'active') || null;
// 方案2：统一转为数字比较
return db.tokens.find((t) => Number(t.kb_id) === Number(kbId) && t.status === 'active') || null;

八、删除能力处理
8.1 现状
KM 官方未提供删除文档的开放接口。kb_client.py 中的 delete_content 方法无法工作。
8.2 替代方案
方案	说明	可行性
隐藏文档	将文档移至隐藏目录或打标签	✅ 可行
版本控制	保留历史版本，标记最新版本	✅ 可行
软删除	添加 deleted 字段，查询时过滤	✅ 可行
联系管理员	通过 KM 管理后台删除	⚠️ 需人工
8.3 建议实现
在 KM-API 层实现软删除：
// 新增字段
interface ContentRecord {
  contentId: number;
  deleted: boolean;
  deletedAt?: string;
  deletedBy?: string;
}
// 查询时过滤
const activeContents = contents.filter(c => !c.deleted);
