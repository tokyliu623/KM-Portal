## 1. 测试范围

### 1.1 覆盖页面（8 个）

| # | 页面 | 路由 | 核心功能 |
|---|------|------|----------|
| 1 | 仪表盘 | `/` | 统计概览、API 调用趋势 |

| 8 | 诊断工具 | `/diag` | Token 验证、翻译健康检查 |

### 1.2 覆盖工作流（5 个）

| # | 工作流 | 描述 | 涉及页面 |
|---|--------|------|----------|
| W1 | Token 全生命周期 | 创建 → 分配 → 使用 → 撤销 → 删除 | Token 管理、KB 浏览 |
| W2 | 知识库内容消费 | 搜索 → 浏览树 → 预览文档 → 复制内容 | KB 浏览、文档编辑 |
| W3 | Skill 创建与导出 | 填写表单 → 触发翻译 → 生成 zip → 下载安装 | Skill 生成 |
| W4 | 运营数据洞察 | 查看 KPI → 趋势分析 → 手动修正 → 健康度评估 | 运营效果分析 |
| W5 | 诊断与问题定位 | Token 诊断 → 翻译探针 → 错误定位 → 修复建议 | 诊断工具 |

### 1.3 验收维度（4 个）

| 维度 | 说明 |
|------|------|
| UI/UX | 布局、响应式、加载状态、空状态、错误提示 |
| 功能正确性 | API 响应、数据渲染、用户交互、边界条件 |
| 错误处理 | 网络异常、权限不足、资源不存在、超时 |
| 性能/稳定性 | 首屏加载、API 响应时间、并发操作 |

## 2. 测试架构

### 2.1 双层测试模型

```
┌─────────────────────────────────────────────┐
│           Playwright (UI 层)                │
│  模拟真实浏览器操作，验证渲染与交互          │
├─────────────────────────────────────────────┤
│           axios (API 层)                    │
│  直接调用后端接口，验证业务逻辑与数据        │
└─────────────────────────────────────────────┘
```

### 2.2 技术选型

| 层级 | 工具 | 理由 |
|------|------|------|
| UI 测试 | Playwright | 跨浏览器支持、截图对比、内置等待 |
| API 测试 | axios + Jest | 精确控制请求、验证响应结构 |
| 报告生成 | 自定义 Markdown | 与项目文档体系一致 |
| 截图基线 | Playwright screenshot | UI 回归对比 |

### 2.3 测试环境

- **本地开发服务器**：`npm run dev`（前端 Vite + 后端 tsx）
- **上游联调**：KB 34754 (Token: `u-46311a0f761a4d7fac562df2b789c96e`) + wiki.vivo.xyz
- **数据隔离**：测试用独立 KB ID，不污染生产数据

## 4. 问题闭环机制

### 4.1 严重度定义

| 级别 | 定义 | 处理 SLA |
|------|------|----------|
| P0 | 阻塞：核心功能完全不可用 | 立即修复 |
| P1 | 严重：功能部分失效，影响主要流程 | 24h 内修复 |
| P2 | 一般：功能异常但有 workaround | 72h 内修复 |
| P3 | 建议：体验优化、边界优化 | 排期处理 |

### 4.2 问题跟踪

- **问题清单**：`docs/qa/ISSUES.md`
- **每个问题包含**：ID、标题、严重度、复现步骤、预期/实际结果、根因分析、修复状态

### 4.3 回归验证

- 修复后重新运行对应用例
- 截图对比（UI 变更）
- 更新 `docs/qa/E2E-VERIFICATION-REPORT.md` 状态

## 6. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 上游 KB API 不稳定 | 测试失败 | 使用 mock 数据 fallback |
| 翻译 API 超时 | Skill 创建慢 | 设置 10s 超时，失败降级 |
| 跨域问题 | 本地测试受阻 | Vite proxy 配置 |
| 测试数据污染 | 影响生产 | 使用独立测试 KB ID |
